    • 使用Github Desktop配合Gridea创建静态页面博客
    2022-03-12 / 2 min read

    众所周知，Github的访问经常抽风，因此依赖于Git的Gridea便经常发生“同步失败”的情况，在看了一大堆的网页资料之后，我终于找到了正常同步的方法。

    Git的访问有时出问题，导致Gridea经常同步失败，因此可以使用GitHub Desktop承担同步作用。
说明：在实体机内，我安装Git与Gridea就可以在Github没有被DNS污染的情况下正常更新，因此下面的方法基于虚拟机内的Windows 8.1 Enterprise X64下进行。

    准备工作

    安装好需要的软件：
    GrideaGithub DesktopGit

    开始搭建

    1. 注册GitHub账号，创建一个库并开启GitHub Pages功能
    2. 在个人设置中创建token
    3. 在Gridea的“远程”选项卡中填入信息
    4. 点击“同步”按钮测试，若可行，下文可以忽略。
    5. 直接打开你的GitHub页面看看效果吧

    同步失败的替代解决方案

    • 打开GitHub Desktop，登陆你的账号
    • 在右侧找到你的网页所在的库，点击克隆到本地，但不要点击确定
    • 把上述提到的clone目录设定到Gridea文件夹中的“output‘文件夹，然后点击确定git到本地
    • 再次点击Gridea中的同步按钮，便可以在GitHub Desktop中看到变化
    • 添加commit并上传到库中
