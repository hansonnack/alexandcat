使用Github Desktop配合Gridea创建静态页面博客

2022-03-12

众所周知，Github的访问经常抽风，因此依赖于Git的Gridea便经常发生“同步失败”的情况，在看了一大堆的网页资料之后，我终于找到了正常同步的方法。

说明：在实体机内，我安装Git加Gridea就可以在Github没有被DNS污染的情况下正常更新，因此下面的方法基于虚拟机内的Windows 8.1 Enterprise X64下进行
首先安装Github Desktop，登录自己的账号，并将仓库clone到本地，将Gredia文件夹中的output文件夹删除，在本地存储目录中新建这个文件夹。在之后的更新中便可以点击Gridea的“同步”按钮更新本地文件，此时Github客户端会检测到更改，添加commit并push到仓库。
当然如果你的Gridea能够正常同步，那么直接下载Git安装，就可以使用自带的同步功能了。

软件下载地址：
Gridea：https://gridea.dev/
Github Desktop：https://desktop.github.com/
Git：https://git-scm.com/download/win
